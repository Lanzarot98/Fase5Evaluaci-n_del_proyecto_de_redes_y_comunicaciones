Servicios Basados en Redes Convergentes

🌐 1. Servicio de Historia Clínica Electrónica (HCE)

Características: Sistema web centralizado para gestión de historias clínicas con acceso desde múltiples subredes.

Funcionalidades: Almacenamiento seguro de datos médicos, acceso desde consultas, laboratorio y urgencias. Implementado en servidor web 192.168.4.22 (IPv4) y 2001:DB8:CAFE:0010::20 (IPv6).

📧 2. Servicio de Correo Electrónico Institucional

Características: Sistema de comunicación unificado para personal médico y administrativo con dominio @hospital.com.

Funcionalidades: Comunicación interdepartamental, notificaciones médicas, coordinación de turnos. Servidor en 192.168.4.21 (IPv4) y 2001:DB8:CAFE:0010::21 (IPv6).

🌡️ 3. Sistema IoT de Monitoreo Ambiental

Características: Red de sensores para control de temperatura, humedad y calidad del aire en áreas críticas.

Funcionalidades: Monitoreo continuo en urgencias, alertas automáticas, control de climatización. Integrado en subred de administración.

Servicios Convergentes (Continuación)

🔗 4. Servicio de Telemedicina

Características: Plataforma de consultas remotas con videoconferencia, intercambio de estudios médicos y segunda opinión.

Funcionalidades: Conectividad con sede capital vía WAN, consultas remotas, transmisión de imágenes médicas. Red dedicada 192.168.3.0/24 y 2001:DB8:CAFE:0030::/64.

📱 5. Red Inalámbrica para Personal Médico

Características: Infraestructura WiFi segura para dispositivos móviles del personal médico con acceso controlado.

Funcionalidades: Movilidad del personal, acceso a HCE desde dispositivos móviles, comunicación instantánea. DHCP automático y segmentación VLAN.

🔧 Implementación en la Red

Todos los servicios están integrados mediante:

  • Enrutamiento OSPF para conectividad total
  • Segmentación por VLANs para seguridad
  • Servidores DNS para resolución de nombres
  • Firewalls para protección perimetral

Análisis de Amenazas de Seguridad

⚠️ 1. Ransomware Hospitalario

Descripción: Malware que cifra historias clínicas y sistemas críticos.

Servicios afectados: HCE, bases de datos médicas, sistemas de laboratorio.

Prevención: Backups automatizados, segmentación de red, actualizaciones de seguridad, capacitación del personal.

🎯 2. Ataques de Denegación de Servicio (DDoS)

Descripción: Sobrecarga de tráfico para inutilizar servicios críticos.

Servicios afectados: Todos los servicios web, telemedicina, comunicaciones.

Prevención: Firewall con protección DDoS, balanceadores de carga, monitoreo de tráfico, CDN.

🕵️ 3. Acceso No Autorizado a Datos Médicos

Descripción: Intrusión para robar información confidencial de pacientes.

Servicios afectados: HCE, bases de datos, correo electrónico.

Prevención: Autenticación multifactor, cifrado de datos, control de accesos, auditorías de seguridad.

Amenazas y Herramientas de Detección

📡 4. Interceptación WiFi

Servicios afectados: Red WiFi, comunicaciones móviles

Prevención: WPA3, VPN, certificados digitales

🦠 5. Malware en IoT

Servicios afectados: Sistema IoT, control ambiental

Prevención: Segmentación IoT, actualizaciones firmware

Herramientas de Detección de Vulnerabilidades

🔍 1. Nmap (Network Mapper)

Uso: Escaneo de puertos, detección de servicios, mapeo de red. Implementado para auditorías regulares de seguridad.

📊 2. PRTG Network Monitor

Uso: Monitoreo continuo de infraestructura, detección de anomalías, alertas en tiempo real.

🛡️ 3. OpenVAS

Uso: Scanner de vulnerabilidades completo, evaluación de seguridad automatizada, reportes detallados.

Procedimientos de Mantenimiento Correctivo

🚨 Clasificación de Incidentes

🔴 CRÍTICO (0-2 horas)

  • Caída total de HCE
  • Falla en servicios de urgencias
  • Compromiso de seguridad
  • Falla total de conectividad

🟡 ALTO (2-8 horas)

  • Falla parcial de servicios
  • Problemas de rendimiento
  • Falla de equipos redundantes
  • Problemas de telemedicina

🟢 MEDIO (8-24 horas)

  • Fallas menores de conectividad
  • Problemas de impresión
  • Actualizaciones fallidas
  • Problemas cosméticos

3. Implementación de Solución
